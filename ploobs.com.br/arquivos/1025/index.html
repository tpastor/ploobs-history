Billboards e Transparência (XNA 4.0)


Neste post falarei um pouco sobre como implementar billboards cilindricas e esfericas. Usarei HLSL para os Shaders e XNA 4.0 como API grafica (porem os podem ser aplicados em qualquer outra API como DirectX e OpenGL).

No fim, falarei brevemente sobre como corrigir alguns problemas de aliasing que esta técnica pode trazer.

Billboards é uma forma de simular objetos 3D complexos sem se preocupar em renderizar um modelo completo em 3D. A idéia é desenhar uma sprite 2D com uma “foto” do modelo ao invés de rederiza-lo completamente. Se a camera se movimentar para tentar ver o modelo sobre outros angulos, teremos que rotacionar a sprite para ela sempre estar “olhando” para a camera. (isso impede que o jogador perceba que na verdade está vendo apenas uma um imagem 2D)

Obviamente, está técnica não é visualmente convincente quanto renderizar o modelo 3D real, porém ela é muiiiiito mais rápida especialmente quando temos uma grande quantidade de objetos (como arvores, grama, particulas…). O exemplo a ser desenvolvido usa somente a GPU para posicionar (olhar sempre para a camera) a billboard na cena.

Existem dois tipos básicos de billboard:

  • Cilindricas, no qual a sprite pode rotacionar em apenas em um eixo (arvores).
  • Esferica, no qual a sprite pode rotacionar em qualquer eixo (particulas).

Implementacão

Existem literalmente zilhoes de maneiras de implementar billboards, eu vou escolher uma que é facil de entender e razoavelmente rapida.

Antes de comecar a falar sobre os detalhes, recomendo fortemente que o pessoal leia o artigo Introducao a Shader aqui.

Parte XNA

Inicialmente temos que definir o “Quad” (quadrado (=P) que é desenhado na tela onde colocaremos nossa sprite). A imagem abaixo mostra vários “Quads” parcialmente transparentes.

Em cada um destes quads “colaremos” uma textura que simula o modelo 3D, um exemplo de textura que pode ser usada para simular grama é:

Como disse anteriormente, posicionaremos os vertices destes quads no shader. No nosso exemplo um quad será definido por seis vértices (dois triânglos, sem usar indices). A estrutura que guardará os vertices tem o seguinte formato (conteudo do vertex buffer):

 VertexPositionTexture[] billboardVertices = new VertexPositionTexture[positions.Count * 6];

Observe que sabemos a priori a quantidade de quads que teremos (positions.Count), multiplicamos por 6 pois cada quad tem 6 vertices, logo teremos que armazenar espaco para 6 * numero de quads Vertices.
Cada conjunto de 6 vertices (ou dois triangulos) representao um Quad, a imagem a seguir ajuda a compreender como preencheremos cada um dos triangulos. Os numeros mostrados sao as coordenas de textura que queremos em cada um dos vertices afim de possamos “colar” a sprite no quad.

Como faremos o posicionamento no shader (muito mais rápido que na CPU), todos os vértices do QUAD terão a mesma posicão (a do centro do quad em espaco de mundo), eles se diferenciarão apenas na coordenada de textura, conforme mostrado na figura acima. (dois triangulos).

Cada quad é preechido da seguinte maneira: (conteudo do vertex Buffer)

            billboardVertices[i++] = new VertexPositionTexture(position, new Vector2(0, 0));
            billboardVertices[i++] = new VertexPositionTexture(position, new Vector2(1, 0));
            billboardVertices[i++] = new VertexPositionTexture(position, new Vector2(1, 1));

            billboardVertices[i++] = new VertexPositionTexture(position, new Vector2(0, 0));
            billboardVertices[i++] = new VertexPositionTexture(position, new Vector2(1, 1));
            billboardVertices[i++] = new VertexPositionTexture(position, new Vector2(0, 1));

Por exemplo: se queremos 5 quads, devemos ter 5*6 = um vertex buffer com espaco para 30 vertices do tipo VertexPosition Texture. Suponha que o primeiro Quad esteja na posicao 0,10,15, entao definiremos os primeiros 6 espacos do vertex buffer da seguinte maneira:

            billboardVertices[0] = new VertexPositionTexture(new Vector3(0,10,15), new Vector2(0, 0));
            billboardVertices[1] = new VertexPositionTexture(new Vector3(0,10,15), new Vector2(1, 0));
            billboardVertices[2] = new VertexPositionTexture(new Vector3(0,10,15), new Vector2(1, 1));

            billboardVertices[3] = new VertexPositionTexture(new Vector3(0,10,15), new Vector2(0, 0));
            billboardVertices[4] = new VertexPositionTexture(new Vector3(0,10,15), new Vector2(1, 1));
            billboardVertices[5] = new VertexPositionTexture(new Vector3(0,10,15), new Vector2(0, 1));

Repare no atributo position (todos iguais) e no atributo TextureCoordinate (gerando os dois triangulos). Os outros quads seguem a mesma idéia.
Deixo ao cargo do leitor criar o Vertex Buffer (não precisa de VertexDeclaration, pois o VertexPositionTexture é um tipo “pre-construido”, voce pode acessa-lo usando VertexPositionTexture.VertexDeclaration). Qualquer problema avisa =P.

//------- Technique: CylBillboard --------
BBVertexToPixel CylBillboardVS(float4 inPos: POSITION0, float2 inTexCoord: TEXCOORD0)
{
    BBVertexToPixel Output = (BBVertexToPixel)0;

    float3 center = mul(inPos, xWorld);
    float3 eyeVector = center - xCamPos;

    float3 upVector = xAllowedRotDir;
    upVector = normalize(upVector);
    float3 sideVector = cross(eyeVector,upVector);
    sideVector = normalize(sideVector);

    float3 finalPosition = center;
    finalPosition += (inTexCoord.x-0.5f)*sideVector * scale;
    finalPosition += (1.5f-inTexCoord.y*1.5f)*upVector * scale;

    float4 finalPosition4 = float4(finalPosition, 1);

    float4x4 preViewProjection = mul (xView, xProjection);
    Output.Position = mul(finalPosition4, preViewProjection);

    Output.TexCoord = inTexCoord;
    return Output;
}

No pixel shader que os vertices são dispostos da maneira correta. Inicialmente calculamos o centro da billboard pegando o valor de posicao do vértice do triangulo e multiplicando pela matriz World, em seguida calculamos o vetor eye (posicao do vertice – posicao da camera). Queremos que o quad tenha a direcao deste vetor eye (aponte para ele). Como ja temos o Up da billboard (cilindrica tem um eixo fixo), devemos apenas calcular o vetor lado (atraves de um cross product entre o up e o eye) e deslocar os vertices de acordo com sua coordenada de textura (neste caso usamos a coordenada de textura para identificar qual vertice do quad estamos). Não é complicado de entender, basta substituir os valores das coordenadas de textures( (0,0) por exemplo) e ver como que o vertice é posicionado (cima esquerda … ).

Outra questão interessante a ser notada é que no Pixel Shader estou checando se o alpha (neste caso interpretado como nível de transparencia) é menor que uma variavel passada (normalmente algo em torno de 0.8), e se for, o pixel é descartado. (partes transparentes) Esta tecnica (alpha testing) é eficaz (não é perfeita) para corrigir problemas de ordenacão devido ao fato de objetos transparentes e  o DepthBuffer não trabalharem “bem” em conjunto (Veja mais informacoes sobre este problema e outras tecnicas de como resolve-lo aqui). O ponto ruim é que se a textura do billboard não for boa teremos problemas com aliasing. Infelizmente em DirectX 9 e em XNA não existe uma solucão perfeita para esse problema.

O shader para a billboard esferica é:

//------- Technique: CylBillboard --------
BBVertexToPixel CylBillboardVS(float4 inPos: POSITION0, float2 inTexCoord: TEXCOORD0 )
{
    BBVertexToPixel Output = (BBVertexToPixel)0;

    float3 center = mul(inPos, xWorld);    

    float3 upVector = CAMERA_UP;
    upVector = normalize(upVector);

    float3 sideVector;
    float3 right = cross(upVector,forward);
    sideVector = normalize(right);    

    float3 finalPosition = center;
    finalPosition += (inTexCoord.x-0.5f)*sideVector * scaleX;
    finalPosition += (1.5f-inTexCoord.y*1.5f)*upVector * scaleY;       

    float4 finalPosition4 = float4(finalPosition, 1);
    float4x4 preViewProjection = mul (xView, xProjection);
    Output.Position = mul(finalPosition4, preViewProjection);
    Output.TexCoord.x = inTexCoord.x;
    Output.TexCoord.y =  inTexCoord.y;
    return Output;
}

Este é bastante semelhante ao anterior, a principal diferenca é que o upVector agora é o Vetor Up da camera (e voce tera que passa-lo do XNA para o shader).

Nao disponibilizarei codigo fonte porque eu nao fiz um exemplo =P, o codigo mostrado é uma adaptacão do usado na nossa engine PloobsEngine, quem estiver interessado pode baixar o codigo inteiro da engine e estuda-lo (lá voce encontrará diversos outros tipos de tecnicas para desenho de billboards como alpha blend dual pass e Instancing).
