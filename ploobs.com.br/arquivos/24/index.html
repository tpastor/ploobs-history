Entrando no mundo 3D


Vamos então adentrar no mundo 3D, meu objetivo nesse mini tutorial é abordar de forma mais simples possível o desenho de um objeto 3D na tela. Num próximo momento explicitarei mais detalhadamente com respeito à aspectos de projeção e de câmera, no momento o meu foco é fazer vocês colocarem objetos 3D na tela.


Aqui está o código inteiro: Tutorial 3

Vamos por partes.
Primeiramente vamos declarar a entidade que vai armazenar e gerenciar o modelo 3d

Model espada;

Com essa entidade temos informações do modelo, como por exemplo o shader (falarei deles futuramente), dos vértices, texturas e tudo mais, nele não existe a noção de objeto físico, ou seja não tem posição, velocidade, ele só é um ente visual.Por isso declaramos um vetor para armazenar informações de posição para o modelo.

Vector3 modelPosition;

Para inicializar o modelo, fazemos da mesma maneira que com o Texture2D

espada = Content.Load(“..\\Content\\Model\\espada”);

Inicializei a posição em qualquer lugar pois depois setaria uma câmera para olhar pra ele.

Desenhando de fato

No XNA tudo que é desenhado usa um effect, um shader ( agora vou falar como usar, amanhã faço um pequeno tutorial dizendo o que é).
A princípio o model está usando um BasicEffect, um material básico com propriedades como cor e luz…
Nessa parte do código, recuperamos cada objeto dentro do modelo que abrimos e cada material atrelado a ele

foreach (ModelMesh mesh in espada.Meshes)
{
foreach (BasicEffect effect in mesh.Effects)
{
//Propriedades do objeto
effect.World = transforms[mesh.ParentBone.Index] * Matrix.CreateTranslation(modelPosition);
//Propriedades da camera
effect.View = Matrix.CreateLookAt(cameraPosition, Vector3.Zero, Vector3.Up);
effect.Projection = Matrix.CreatePerspectiveFieldOfView(MathHelper.ToRadians(45.0f),
aspectRatio, 1.0f, 10000.0f);
}
mesh.Draw();
}

e temos de configurar o material usado por esse objeto, configurei os parâmetros básicos como o world, view, e projection:

No world eu podia ter facilitado, mas vamos lá né:
effect.World = transforms[mesh.ParentBone.Index] * Matrix.CreateTranslation(modelPosition);

A primeira parte ( transforms[mesh.ParentBone.Index] ) nos dá informações relativas aos objetos dentro do modelo, por exemplo um modelo em que tivermos três caixas, cada uma tem uma posição relativa ao (0,0,0) e é necessário que
recuperemos tais valores pois se não corremos o risco de deixarmos todos os objetos encavalados.
A segunda parte ( Matrix.CreateTranslation(modelPosition) ) é a posição em que queremos colocar o objeto. Você deve estar se perguntando agora, porque essa Matrix.CreateTranslation ? É que existem para cada tipo de ação, uma matriz característica que a caracterisa, e como ninguém tem obrigação de saber disso tudo, a classe Matrix traz funções utilitárias que cria essas matrizes.

No View
usei uma das funções utilitárias do Matrix para criar um LookAt que deve dizer basicamente onde está a câmera, pra onde ela está olhando e a sua orientação, ou seja, se está de ponta cabeça, de lado.

No projection entra um conceito importante, que é o fato de vermos e vivermos em um mundo 3d e nosso monitor ser plano, temos de criar uma projeção de tal forma que aparente que estamos vendo a mesma coisa.

Essa imagem que retirei do http://www.3dgameprogramming.net/ explica bem essa função, o primeiro parâmetro é o field of view, que é o ângulo que a câmera vizualiza, note que quanto maior for esse ângulo, parece que o objeto está mais longe pois tem que caber na tela, varie esses parâmetros até perceber as alterações.
O aspect ratio é a proporção entre altura e largura, normalmente são os famosos 4:3 que usamos nos monitores mas você pode colocar a proporção que quiser e distorcer a imagem como quiser.
Os dois últimos parâmetros são o near plane e o far plane da image
m, lembrando que só é desenhado o que fica entre o nearplane e o farplane, de forma geral o que é desenhado é o que fica dentro dessa pirâmide da imagem.

O meu intuito foi fazer vocês desenharem um modelo 3d na tela de maneira simples, espero ter sido simples
