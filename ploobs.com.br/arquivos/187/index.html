PloobsEngine “First Blood”


ESTE ARTIGO É BASTANTE ANTIGO E ESTA MUITO DESATUALIZADO,  VISITE http://ploobs.com.br/?page_id=1381.

Começamos agora uma série de posts sobre a PloobsEngine, uma Engine gráfica de tempo real.

Iremos abordar assuntos como a arquitetura, decisões de projeto tomadas ao longo do processo, bem como mostrar exemplos de uso das features ao longo do tempo.

A PloobsEngine foi implementada sob o XNA ( Framework criada pela Microsoft para desenvolvimento de jogos para Windows, Xbox, Zune e WindowsPhone ), porém a estrutura é desacoplada o suficiente para ser implementada sob qualquer API gráfica.

Imagem de debug com SSAO e Deferred Rendering

Uma Engine é constituída de diversos módulos, cada uma desempenhando um papel expecializado, entretanto vamos nos focar no módulo renderizador por enquanto.

Nesse post vamos mostrar as principais features da nossa engine e futuramente colocaremos exemplos com imagens, videos e Demos.


Features

Audio

  • 3D sound (static and dynamic)
  • 2D sound
  • Music Player

GUI

  • HUD system  ON PROGRESS

Gráficas

  • Deferred Rendering
  • Forward Rendering
  • Deferred and Forward working together
  • Hard Shadows ON PROGRESS
  • BumpMaps
  • SpecularMaps
  • Lights ( PointLight (possibilita mais de 400 na mesma cena),  SpotLight, DirectionalLight)
  • SSAO
  • AntiAliasing
  • PostEffects( ToonShading, Blur, Noise, Wiggle, Circular Glow, DOF,Bloom, Radial Blur, Negative, Black & White)
  • WaterShading
  • EnvironmentMap ( Static and Dinamic )
  • Planar Reflexion
  • Particle System ( Buid in Smoke, Fire and projectiles )
  • Transparency on Deffered
  • Billboard
  • Movies
  • Advanced Culling ( Objects and Lights )

Física

  • JigLibx integration
  • Editor de collision skin
  • Calculo automático de collision skin
  • Object Types ( Box, Sphere, Plane, Triangle, Capsule, Car, Ghost, HeightMap )
  • Trigger System

Controle

  • Troca de mensagens entre entidades
  • Sistema de Input
  • Suporte a Componentes externos
  • Picking Object
  • Resource Control
  • Event System
  • Simple Extensible Factory System
  • Scene Management

Cameras

  • Quake style
  • 3rd Person
  • 1st Person
  • Static
  • Follow path
  • Camera Interpolators

Animação

  • BoneAnimation integrada com Deffered  ( XNAanimation )

Profilling

  • FPS
  • MethodCall
  • Save to File
  • Graph on the Screen
  • Debug ( Physics, Lights, RenderTargets )
  • LogFile

IA

  • Sensors
  • Actuators
  • Path Finding
  • State Machine
  • Behaviors ON PROGRESS
  • Genetic Algorithms
  • Neural Network
  • WayPoint System ( automatic generation)
  • Agent abstraction

Serialization

  • Scene Serialization to XML File
