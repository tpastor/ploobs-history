Artemis and PloobsEngine working together


All right now it is time to go deeper and solve some problems.

  • How to sync between this two world ?
  • How to create the artemis objects dinamically and persist in ploobsengine world ?
Developing the TechDemo, we face some chalanges. One of the most was to connect Artemis with PloobsEngine without been to much intrusive for either side.

Architecture

To be coherent with both systems, we decided to create a component to handle the collision information and a system to treat it every time it ocurs.

Binding the two Worlds

Both Artemis and Ploobs have entity controlling in its worlds,  but if we managed to use then together, they must work as if it were the same, so retrieving one we can have acess to the other.

In this example i show the creation of an entity

  public Entity CreatePlayerMissile(EntityWorld world)
        {
            Entity missile = world.CreateEntity();
            GamePool pool = (GamePool)world.GetPool();
            missile.SetGroup("BULLETS");

            string modelname = "PlayerBullet";
            I2DObject ob = PloobsObjectFactory.Instance().GetObject(modelname);

            FarseerObject b = (FarseerObject)ob.PhysicObject;
            b.Body.OnCollision += new OnCollisionEventHandler(Body_OnCollision);
            missile.AddComponent(new PloobsIObject(ob, modelname));

            Initialize(missile);
            return missile;
        }
}

The line

e.SetUniqueId(obj.Ploobsobject.GetId());

Shows the binding between worlds.

It is important to say that we are using here a non canonical aproach of the entitysystem, creating a component name PloobsIObjectComponent that handles an entire PloobsObject, containing, physics and the visual representation, we made this choice for two reasons, first to save components and to make easier the physics manipulation as the physics manipulation itself it is done by the physic engine and not by any system.

Collision Component

To handle collision, we must have a component witch handles its informations

 public class Collision : Component
    {

        Vector3 force;
        float collideAt;
        Entity hittedby;

        public Entity Hittedby
        {
            get { return hittedby; }
            set { hittedby = value; }
        }

        public Collision(Entity hit, Vector3 force, float collide)
        {
            this.hittedby = hit;
            this.force = force;
            this.collideAt = collide;
        }

    }


 


Handling Collision


Not to handle the collision itself, we must have a event in our physic engine that handles collision. In the example i am using Farseer, which is the 2D physics engine middleware in ploobs engine.


b.Body.OnCollision += new OnCollisionEventHandler(Body_OnCollision);


The line above shows how to set the event.


 public bool Body_OnCollision(FarseerPhysics.Dynamics.Fixture fixtureA, FarseerPhysics.Dynamics.Fixture fixtureB, FarseerPhysics.Dynamics.Contacts.Contact contact)
        {

            Entity e = GetEntityByUniqueId(fixtureA.Body.BodyId);
            Entity hitobj = GetEntityByUniqueId(fixtureB.Body.BodyId);

            if (e != null && hitobj != null)
            {

                Collision collisionA = new Collision(hitobj, Vector3.Zero, contact.TOI);
                e.AddComponent(collisionA);
                e.Refresh();

                Collision collisionB = new Collision(e, Vector3.Zero, contact.TOI);
                hitobj.AddComponent(collisionB);
                hitobj.Refresh();

                return true;
            }
            else
            {
                return false;
            }

        }


Systems Architecture


Now we must treat this handled collisions, a simple but good idea its to create your collisionsystems to treat sound, particles and others things you´d rather and at the end a cleancollisionsystem that removes all the collision components, so in the next update cycle of Artemis another different collisions
