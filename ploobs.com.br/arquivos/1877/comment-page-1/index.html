Recovering VertexBuffer Data for Debug Purpose under DirectX 11


Sometimes we want to see exactely what our Vertex/Geometry/Hull/Domain Shaders are doing. One way is to use those debug softwares like Pix or Nvidia Parallel Nsight. The other way is recover the output data (these shader output) from GPU and write them to the console.


In my opinion, this second approach is faster (in the sense of finding what is going on —  not talking about performance here =P).
The idea behind it is very simple, we send the geometry to the pipeline, process them and after we stream out (DirectX 10 funcionality) the data produced to a buffer. In CPU we just read it =P

To be able to read it, we need to create the following struct:

ID3D11Query*						g_pDeviceStats ;

Initialize it with null.
Then, surround your draw function with these Directx calls (Begin and End functions)

#if defined _DEBUG && defined DEEPDEBUG
	ImediateDeviceContext->Begin(g_pDeviceStats);
#endif
	if(firstTime)
	{
		ImediateDeviceContext->Draw(numParticles,0);
		firstTime = false;
	}
	else
	{
		ImediateDeviceContext->DrawAuto();
	}
#if defined _DEBUG && defined DEEPDEBUG
	ImediateDeviceContext->End(g_pDeviceStats);
#endif

To recover the Data, just use the following command:

D3D11_QUERY_DATA_PIPELINE_STATISTICS stats;
	while( S_OK != EngineStuff::EImediateDeviceContext->GetData(g_pDeviceStats, &stats, g_pDeviceStats->GetDataSize(), 0 ) );

	EngineStuff::EImediateDeviceContext->CopyResource( g_pStagingStreamOutBuffer, mStreamOutVB );
	D3D11_MAPPED_SUBRESOURCE data;
	if( SUCCEEDED( EngineStuff::EImediateDeviceContext->Map( g_pStagingStreamOutBuffer, 0, D3D11_MAP_READ, 0, &data ) ) )
	{
		ParticleVS *pRaw = reinterpret_cast< ParticleVS* >( data.pData );	///nunca achei q ia usar reinterpret_cast na vida !!!!

		for(int i  = 0 ; i < stats.GSPrimitives; i++)
		{
			char* arr = new char[100];
			sprintf(arr,"PARTICLE %d \n",i);
			OutputDebugString(arr);
			sprintf(arr,"Type : %d \n",pRaw[i].type);
			OutputDebugString(arr);
			sprintf(arr,"Age : %f \n",pRaw[i].age );
			OutputDebugString(arr);
			sprintf(arr,"Size : x %f y %f \n",pRaw[i].sizeW.x ,pRaw[i].sizeW.y );
			OutputDebugString(arr);
			sprintf(arr,"InitialPosW : x %f y %f z %f \n",pRaw[i].initialPosW.x ,pRaw[i].initialPosW.y ,pRaw[i].initialPosW.z  );
			OutputDebugString(arr);
			sprintf(arr,"InitialVelW : x %f y %f z %f \n",pRaw[i].initialVelW.x ,pRaw[i].initialVelW.y ,pRaw[i].initialVelW.z  );
			OutputDebugString(arr);
			OutputDebugString("\n");
			delete  [] arr;
		}
		OutputDebugString("------------------------\n");
		EngineStuff::EImediateDeviceContext->Unmap( g_pStagingStreamOutBuffer, 0 );
	}

Pay attention to the line:

		ParticleVS *pRaw = reinterpret_cast< ParticleVS* >( data.pData )

Here you cast the vertex buffer data to the right format (the one you specified as output from your shader).
This approach helped me lots of times, i hope it helps someone else =P
