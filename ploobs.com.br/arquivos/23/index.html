SVN – Entendendo e Usando


O SVN ou subversion é um sistema para controle de versões, muito utilizado em projetos onde muitas pessoas colaboram de forma simultânea. O controle é feito, como o nome sugere por subversões, onde cada uma pode ser recuperada no momento que for necessário…

Alguns pontos básicos a respeito da manipulação de um svn segundo o meu ponto de vista, os comandos que eu mais usei e que acredito que para iniciar são mais do que necessários e algumas dicas importantes que fui percebendo ao fazer tudo errado
Pontos importantes:
  • Basicamente as duas pastas mais importantes de um repositório ( é assim que é chamado o lugar onde os arquivos ficam) são:

– Trunk: Pasta principal, como numa árvore o tronco, nele fica o projeto que está de fato sendo atualizado, o projeto principal

– Branch: Um ramo digamos assim, nele fica um projeto secundário, um exemplo muito bom dele é por exemplo ao fazermos um projeto, decidirmos que uma versão não está muito boa e vamos dividir a nossa equipe em duas, uma vai ficar dando suporte a essa e outra parte da equipe vai começar de novo.
  • Uma coisa importante é que, uma vez adicionado um arquivo no repositório, nunca mais o arquivo será excluído, porque ele estará em outra versão, por isso é sempre bom pensar umas 3 vezes antes de adicionar algo. (isso é algo que ainda estou aprendendo)
Comandos básicos

  • Checkout : Traz um repositório ou parte dele como cópia local para o seu pc
  • Update : Baixa atualizações feitas desde o último update
  • Commit: Envia mudanças feitas por você.
  • Add: adiciona um arquivo ao repositório
  • Remove: remove um arquivo ( da versão atual)
  • Rename: renomeia um arquivo
Em ambientes Unix esses comandos são dados de forma svn [comando] ; exemplo: svn update;
Outros comandos
Conflitos
Com respeito a conflitos posso dizer que você vai ter muitos problemas com eles.Conflitos ocorrem quando dois ou mais usuários comitaram o mesmo arquivo com conteúdo diferente então são automaticamente criados outros arquivos, um com o nome do arquivo e com extenção da versão do arquivo ou arquivos que estão em conflito, um arquivo .mine que é a sua versão, e um com a extensão normal, é nesse arquivo que ficam mesclados todos os outros de forma que com programas de resolução de conflitos podemos ver que partes estã com problemas e quais são eles. Essa é a parte mais difícil, resolver os conflitos, mas isso é uma arte que só o tempo e o google podem te ensinar.
Usando por exemplo o Tortoise esse trabalho ( exceto o de resolver conflitos) fica bem mais intuitivo, clicamos na pasta ou arquivo onde queremos realizar algum comando.
Por exemplo para criar uma cópia local do repositório do Ploobs escolhemos uma pasta
e selecionamos svn checkout
