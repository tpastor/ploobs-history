Developers Diary – Using Artemis as Entity System


As i said in the previous post we choose to use Artemis as the entity system. In the post in the reference you can read more about Artemis.

The approach of entity, component and system give us a so big freedom that for example after programming the game, creating our component and systems, we can easily migrate the game to others languages and even other platforms, just by changing the rendering and input stuffs.

The basic idea of the systems is :

Entity : A container of components, you can make a metaphor of a bag full of items, the entity represents the object itself but it contains components, but it isn´t nothing but this.

Component : You can interpret a component as a behavior. A litle example can say more than words.

Think in a car. We can say that is a component:

  • The ability to “walk”
  • The acceleration
  • The speed
  • The position
  • The material
Finally, all the characteristics of an object can be expressed in components.
But a component only have informations and not logic, NEVER logic.
System: And if i want to make my car became green if its speed exceed 100 units , you make this in an system. And there is the beauty behind this systems. Creating a system, we create a manner to treat a situation so every entity that have for example the material and speed component will behave the same way.
We have a litle problem to sync ploobs engine and Artemis to work together, but that i will keep to the next post.
