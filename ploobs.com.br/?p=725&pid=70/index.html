PloobsEngine for XNA 4.0 Alpha Version Released


We are very proud to announce that the XNA 4.0 port of the PloobsEngine is almost complete. We are now in an open alpha test phase. (BETA VERSION RELEASED, CHECK HERE)

The architecture was a complete re-think and re-write, now it is more user friendly and easy to use. Not all of the PloobsEngine XNA 3.1 version funcionalities were ported for this alpha version.

 

The main features (some are new):

  • Forward Rendering supporting Post Effects and Components integration (works with skybox for example ).
  • Deferred Rendering
  • 3D and 2D Sound
  • Physics Integrated (Bepu API)
  • Animation Integrated (XnaAnimation API)
  • Dynamic Water and Ocean with waves
  • Dynamic Lights (Spot, Point and Directional)
  • BumpMaps, SpecularMaps,Glow Maps and Paralax Mapping
  • Terrain with HeightMaps and Multitexture
  • Soft Dynamic Shadow (Shadow Mapping and Cascade Shadow Mapping) with filtering
  • Deferred with Antialiasing (We have three different algorithms now)
  • Post Effects collection: ToonShading, Blur, Noise, Wiggle, Depth Of Field, Bloom, HDR, Radial Blur, Negative, Black and White, SSAO, Color Correction, Gama Correction, Saturation, Contrast, Fog, Ambient Scattering and much more …
  • Extensible Particle System (using http://www.xnaparticles.com/)
  • Transparency (deferred and forward)
  • Environment Mapping
  • Skybox
  • Billboards (normal and instanced)
  • Animated Billboards and Textures
  • Video Player Embedded
  • Trigger System
  • Message System
  • Picking System (changed)
  • Task Component (easy way to do mult-threading)
  • Input Control System (new features added)
  • Resource Management System
  • Screen Manager System (lots of extra features added)
  • Gui System using a custom version of Neoforce (http://neoforce.codeplex.com/)

The only Documentation of the alpha release is the examples demos (of course the final version will have a proper documentation =P). You can find the old documentation here, things changed a bit but still worth reading. Any doubts, question, pls use our forum.

This release consists of:

  • DLL package containing the PloobsEngineDebug.dll, PloobsEngineDebug.xml and the PloobsEngineContentDebug.dll
  • Introduction Demos package
  • Advanced Demos package
  • Visual Studio 2010 Templates to help people using the engine

The Introduction Demos is a collection of basic samples showing how use the most basic funcionalities of the engine. This project contains:

  • A Basic Engine Setup (For Deferred Rendering and Forward Rendering)
  • InputSystem basic usage
  • Deferred Lights basic usage
  • Setup and extend the Physic World
  • Dealing with Cameras
  • Dealing with trigger, message system and picking system

The advanced Demos is also a collection of demos that shows some more advanced features like:

  • Materials Usage
  • Shadow
  • Water
  • Billboards
  • Particles
  • Terrain
  • Sound
  • Animation
  • Our Loader (integrated with 3DS Max)
  • Gui system
  • PostEffects (we only  used the SSAO for the sample)
  • 3DS Max Loader (load scenes exported using our Exporter plugin)

Some of the features like Culling system were ported, but there is no demo showing how to use it yet.

Using the Visual Studio 2010 Templates

As said, this version provides three templates to be used in visual studio 2010. The first one is a complete Engine Setup project (which includes the Engine Dlls ) , the second and the third are IScreens setups for Forward and Deferred Rending. The package can be downloaded in the Downloads section below.

To install the templates in your Visual Studio you just need to extract the downloaded templates file in a folder of your preference, after you take the files PloobsEngine Deferred Screen Template.zip and PloobsEngine Forward Screen Template.zip and put in your Visual Studio 2010 item templates folder (normaly here C:\Users\USER_NAME\Documents\Visual Studio 2010\Templates\ItemTemplates ) and take the file PloobsEngine Project Template.zip and put it your Visual Studio 2010 projects templates folder (normaly here C:\Users\USER_NAME\Documents\Visual Studio 2010\Templates\ProjectTemplates ).

When you start the Visual Studio 2010 you will be able to see the the PloobsEngine Project template (under C# projects tab) when clicking in New Project and be able to see the PloobsEngine IScreens (under C# projects tab) when clicking in New Item.

Using the Ploobs Exporter Plugin

If you can read portuguese (if you cant, try using the translation), i advise you to check this post. The Plugin is a MaxScript file that reads your Max scene and export a XML file describing it. The exported XML does not contains models informations (vertices and textures) , for this you need a .X exporter, i strongly suggest the http://www.kwxport.org/ (We have a Portuguese post about how to use it).

Our plugin exports:

  • Standard Material including Bump, Self Luminance, diffuse, specular maps
  • Lights Omni and Spot
  • Cameras
  • Dummies
  • Constraint (Point to Point)
  • Multi-Objects (you export just one .x and access each object inside the engine)

To run the exporter in 3DS Max, go to MaxScript menu and choose Run Script, choose our PloobsLevelExporter.ms and run it, the result will be a xml file.

The engine recieves a .X (exported using Kwxport for example), the textures and the XML of the scene, and load them. The Advanced Demos Project have an example showing how to load a scene exported this way.  Any problem using the exporter ? Use our forum =P.

The last version of the exporter can be downloaded in the Download section below.

PloobsEngine Xna 4.0 Roadmap

We are planning to collect user feedbacks about this version to improve stability and usability. During the alpha period we will keep porting the remaining features and correcting eventual bugs, people will always be able to get night builds on our port repo.

Despite all of our features works on Xbox, we did not tested it enough in this platform, so we preferred to just release the alpha for PC, but the engine will support XBOX and Phone7 (with reduced features) when fully released.
